Key Design Decisions:
  • React SPA for responsive, modern UI with real-time updates
  • BullMQ job queue for scalable background processing
  • WebSocket for live progress tracking (no polling)
  • Playwright browser pool for reliable screenshot capture
  • Redis caching for MCP responses (24-hour TTL)

🧩 Module Breakdown

Backend Modules

1️⃣ Audit Engine

Purpose: Orchestrate 5-phase audit process with state management

  • Phase 1: Pre-Audit Research (14 steps)
  • Phase 2: Browser Testing (20 steps)
  • Phase 3: Scoring (10 areas)
  • Phase 4-5: Deliverables (3 files)

Files: audit-engine.ts, phases/phase1-5.ts

2️⃣ MCP Proxy Service

Purpose: Wrap 5 MCP servers with REST APIs, handle retries, caching

  • BuiltWith API (7 endpoints)
  • SimilarWeb API (14 endpoints)
  • Yahoo Finance API (8 tools)
  • Perplexity WebSearch

Files: mcp-proxy/builtwith.ts, similarweb.ts, etc.

3️⃣ Browser Pool Service

Purpose: Manage Chrome instances, screenshot capture, WAF handling

  • Pool of 10 Chrome instances
  • Screenshot persistence to S3
  • CAPTCHA detection & user handoff
  • WAF recovery with retries

Files: browser-pool.ts

4️⃣ Job Queue (BullMQ)

Purpose: Manage long-running audit jobs, retry failed steps

  • Redis-backed queue
  • 3 retry attempts with exponential backoff
  • Concurrency: 5 audits at once
  • Job lifecycle events

Files: jobs/audit-queue.ts

5️⃣ WebSocket Server

Purpose: Real-time progress updates to frontend

  • Socket.IO server
  • Subscribe/unsubscribe per audit
  • Broadcast progress, phase completion, errors
  • Auto-reconnect handling

Files: websocket/server.ts

6️⃣ API Gateway

Purpose: REST API with auth, validation, rate limiting

  • Express.js server
  • Okta SSO authentication
  • JWT token validation
  • Request/response validation

Files: api/routes/*.ts, controllers/*.ts

Frontend Modules

1️⃣ Dashboard Page

Purpose: Central hub for all audits

  • Audit list with filters
  • Status badges (completed, in-progress, failed)
  • Search & sorting
  • Quick actions (view, download, star)

Files: pages/Dashboard.tsx

2️⃣ Create Audit Wizard

Purpose: 4-step guided form

  • Step 1: Enter domain
  • Step 2: Select phases
  • Step 3: Custom queries (optional)
  • Step 4: Review & launch

Files: pages/CreateAudit.tsx

3️⃣ Execution Monitor

Purpose: Real-time progress tracking

  • Overall progress bar
  • Phase-by-phase breakdown
  • Live logs (WebSocket-driven)
  • Pause/cancel controls

Files: pages/ExecutionMonitor.tsx

4️⃣ Audit Details

Purpose: Post-completion drill-down

  • Tab 1: Overview (KPIs, score)
  • Tab 2: Research Data (12 scratchpad files)
  • Tab 3: Findings (screenshot gallery)
  • Tab 4: Deliverables (PDF viewer + downloads)
  • Tab 5: Settings (re-run, fact-check, export)

Files: pages/AuditDetails.tsx

5️⃣ API Client

Purpose: Axios-based API wrapper

  • REST endpoints for audits, scratchpad, screenshots
  • Auth token injection
  • Error handling & retries

Files: api/client.ts, api/audits.ts

6️⃣ State Management

Purpose: Zustand stores for global state

  • Audit store (list, current audit)
  • User store (profile, permissions)
  • UI store (modals, toasts)

Files: stores/audit-store.ts

⚙️ Technology Stack

Layer Technology Why We Chose It
Frontend React 18 TypeScript Mantine UI Industry standard, mature ecosystem, excellent TypeScript support
State Mgmt Zustand Socket.IO Client Lightweight alternative to Redux, real-time WebSocket updates
Backend Node.js 20 Express.js TypeScript Fast, scalable, TypeScript support, large community
Job Queue BullMQ Socket.IO Redis-backed queue, reliable job processing, WebSocket for live updates
Browser Playwright More reliable than Puppeteer, better WAF handling, cross-browser support
Database PostgreSQL 15 Redis 7 Relational data with JSONB support, Redis for caching and job queue
Storage AWS S3 Object storage for screenshots, PDFs, scratchpad files
Auth Okta SSO JWT Enterprise SSO, secure token-based authentication
APIs BuiltWith API SimilarWeb API Yahoo Finance Perplexity AI Tech stack, traffic data, financial data, web search

Database Schema (8 Tables)

users (id, okta_id, email, name, team, role) audits (id, domain, company_name, status, progress_pct, overall_score, config) scratchpad_files (id, audit_id, file_name, content, data_quality) screenshots (id, audit_id, file_name, query, severity, storage_url) deliverables (id, audit_id, type, storage_url, brand_score) execution_logs (id, audit_id, timestamp, phase, message) mcp_calls (id, audit_id, server_name, endpoint, response_data, latency_ms) templates (id, name, created_by, config, usage_count)

🗓️ Implementation Roadmap

Phase 1: MVP (8 weeks) Pending

Goal: Core audit flow with sequential execution (no agent teams yet)

  • Week 1-2: Infrastructure (DB, Redis, S3, Express API, React boilerplate)
  • Week 3-4: Phase 1 Research (MCP proxies, scratchpad storage, WebSocket)
  • Week 5-6: Phase 2 Browser Testing (Playwright pool, 20-step suite, screenshots)
  • Week 7: Phase 3-5 (Scoring, deliverable generation, PDF rendering)
  • Week 8: UI Polish (dashboard, wizard, execution monitor, deliverable viewer)
Deliverable: Working platform with sequential audit execution (~60 min runtime)

Phase 2: Agent Teams (4 weeks) Pending

Goal: Parallel execution for 2x speed improvement

  • Week 9-10: Agent Teams Coordinator (Wave 1-4 orchestration)
  • Week 11-12: Optimization (MCP caching, browser pool scaling)
Deliverable: 35-minute runtime (down from 60+ minutes in MVP)

Phase 3: Polish & Scale (4 weeks) Pending

Goal: Production-ready with enterprise features

  • Week 13-14: Fact-Check Integration (7-dimension validation)
  • Week 15-16: Enterprise Features (templates, admin dashboard, Salesforce integration)
