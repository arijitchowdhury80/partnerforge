🏗️ System Architecture

Key Design Decisions:
  • React SPA for responsive, modern UI with real-time updates
  • BullMQ job queue for scalable background processing
  • WebSocket for live progress tracking (no polling)
  • Playwright browser pool for reliable screenshot capture
  • Redis caching for MCP responses (24-hour TTL)

🧩 Module Breakdown

Backend Modules

1️⃣ Audit Engine

Purpose: Orchestrate 5-phase audit process

Phase 1: Pre-Audit Research (14 steps)
  • 1. Company Context (overview, revenue, employees)
  • 2. Tech Stack (BuiltWith 7 endpoints)
  • 3. Traffic (SimilarWeb 14 endpoints)
  • 4. Competitor Identification
  • 5. Test Queries Generation
  • 6. Competitor Search Analysis
  • 7. Strategic Angle Mining
  • 8. Hiring Signal Detection
  • 9. Financial Synthesis + ROI (Yahoo Finance)
  • 10. Trigger Event Synthesis
  • 11. Vertical Matching
  • 12. Investor Intelligence (SEC/earnings)
  • 13. Deep Hiring + Buying Committee
  • 14. ICP-to-Priority Mapping
Phase 2: Browser Testing (20 steps)
  • Core Tests (12):
  • 1. Homepage observation
  • 2. Empty state test
  • 3. SAYT (Search As You Type)
  • 4. Full results page
  • 5. Typo tolerance
  • 6. Synonyms handling
  • 7. No results handling
  • 8. Non-product content search
  • 9. Intent detection
  • 10. Merchandising consistency
  • 11. Federated search
  • 12. Mobile experience
  • Algolia Value-Prop Tests (8):
  • 13. Semantic/NLP search
  • 14. Dynamic facets
  • 15. Popular searches
  • 16. Dynamic categories
  • 17. Personalization
  • 18. Recommendations
  • 19. Merchandising rules
  • 20. Analytics visibility
Phase 3: Scoring (10 areas)
  • 1. Search-as-You-Type (SAYT)
  • 2. Query Understanding (typos, synonyms)
  • 3. Results Quality
  • 4. Faceting & Filtering
  • 5. Federated Search
  • 6. Mobile Experience
  • 7. Semantic/NLP
  • 8. Dynamic Facets & Personalization
  • 9. Recommendations & Merchandising
  • 10. Search Intelligence
Phase 4-5: Deliverables (3 files)
  • 1. Search Audit Book (PDF, 40-50 pages)
  • → Executive summary, findings, screenshots
  • → Industry stats, competitor analysis
  • 2. AE Pre-Call Brief (Markdown, 10-15 pages)
  • → Buying committee, talking points
  • → Pain points, ROI opportunities
  • 3. Strategic Signal Brief (Markdown, 3-5 pages)
  • → LLM-ready format for downstream
  • → Trigger events, hiring signals

2️⃣ MCP Proxy Service

Purpose: Wrap 5 MCP servers with REST APIs

  • ✓ BuiltWith API (7 endpoints)
  • ✓ SimilarWeb API (14 endpoints)
  • ✓ Yahoo Finance API (8 tools)
  • ✓ Perplexity WebSearch

3️⃣ Browser Pool Service

Purpose: Manage Chrome instances

  • ✓ Pool of 10 Chrome instances
  • ✓ Screenshot persistence to S3
  • ✓ CAPTCHA detection & handoff
  • ✓ WAF recovery with retries

4️⃣ Job Queue (BullMQ)

Purpose: Manage long-running audit jobs

  • ✓ Redis-backed queue
  • ✓ 3 retry attempts with backoff
  • ✓ Concurrency: 5 audits at once
  • ✓ Job lifecycle events

5️⃣ WebSocket Server

Purpose: Real-time progress updates

  • ✓ Socket.IO server
  • ✓ Subscribe/unsubscribe per audit
  • ✓ Broadcast progress, errors
  • ✓ Auto-reconnect handling

6️⃣ API Gateway

Purpose: REST API with auth

  • ✓ Express.js server
  • ✓ Okta SSO authentication
  • ✓ JWT token validation
  • ✓ Request/response validation

Frontend Modules

1️⃣ Dashboard Page

Central hub with audit list, filters, search, and quick actions

2️⃣ Create Audit Wizard

4-step guided form: Domain → Phases → Queries → Review & Launch

3️⃣ Execution Monitor

Real-time progress with WebSocket updates, live logs, pause/cancel

4️⃣ Audit Details

5 tabs: Overview, Research Data, Findings, Deliverables, Settings

5️⃣ API Client

Axios-based wrapper with auth token injection and error handling

6️⃣ State Management

Zustand stores for audits, user profile, and UI state
