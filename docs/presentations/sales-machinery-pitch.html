Executive Summary

The Opportunity

We have access to thousands of accounts using our partner technologies (Adobe, Amplience, Spryker). But which ones should we prioritize? This proposal introduces a data-driven cascading funnel that systematically identifies the highest-value accounts and assigns the right sales plays and campaigns.

The Problem

Traditional lead scoring treats all accounts equally. A SaaS company with high traffic scores the same as a Fashion retailer with high traffic. But our data shows Fashion delivers 65% of all proof points. They are NOT equal.

The Solution

A four-layer cascading funnel that filters accounts through:

  1. Partner Tech Cohort - What tech stack do they have?
  2. ICP Filter - Do they match our Ideal Customer Profile?
  3. Sales Play Intersection - What plays can we run?
  4. GTM Campaign - How do we engage?

The first filter classifies accounts by their tech stack. Accounts with multiple partner technologies signal enterprise-grade investment and higher deal potential.

Layer 2: ICP Filter

Once we identify JACKPOT accounts, we apply the ICP filter to further prioritize based on vertical fit, competitor displacement opportunity, and other dimensions.

2
ICP Scoring Formula
ICP_Score = (
    Vertical × 0.25 +      // Fashion/Grocery = 100
    Competitor × 0.20 +    // Elastic/Solr = 100
    Size × 0.15 +          // Enterprise = 100
    TechStack × 0.15 +     // SFCC/Magento = 90-100
    Persona × 0.15 +       // CTO access = 100
    Geography × 0.10       // US/UK = 100
)

Tier Score Range Characteristics Action

Tier 2 70-84 General Retail + Mid-Market + Known competitor Nurture pipeline
Tier 3 50-69 SaaS/B2B + Growth stage Long-term
Tier 4+ <50 Weak fit Deprioritize/Exclude

Layer 3: Sales Play Intersection

For ICP Tier 1 accounts, we apply the Sales Play filter. Accounts can be tagged with multiple plays, and intersections are the highest priority.

The Three Sales Plays

Play Code Definition Source
Tech Partner S1 Uses partner tech (Adobe, Amplience, Spryker) BuiltWith, Crossbeam
Target List S2 On marketing/ABM target list CSV upload, Marketing
SI Connected S3 Has relationship with SI partner (EPAM, Valtech) Crossbeam, Manual

The Intersection Model

Key Insight:

Accounts tagged with multiple plays are validated from multiple angles. The more plays, the higher the confidence.

Intersection Plays Priority Multiplier Campaign Type
Triple Play S1 + S2 + S3 JACKPOT 1.5x ABM 1:1
Tech + Target S1 + S2 HIGH 1.25x ABM 1:Few
Target + SI S2 + S3 HIGH 1.25x ABM 1:Few
Tech + SI S1 + S3 MEDIUM 1.25x ABM 1:Few
Single Play S1, S2, or S3 STANDARD 1.0x ABM 1:Many

Final Priority Calculation

Final Priority = ICP Score × Play Multiplier

Example (Under Armour with S1+S2):
Final Priority = 87.75 × 1.25 = 109.7

Example (Costco with S1+S2+S3):
Final Priority = 92.0 × 1.5 = 138.0

Implementation Plan

Phase 1: Data Foundation (Week 1-2)

  • Query BuiltWith for all partner tech accounts
  • Classify each into cohort (JACKPOT/HIGH/MEDIUM/STANDARD/LOW)
  • Store cohort classification in database
  • Build dashboard showing cohort distribution

Phase 2: ICP Scoring (Week 2-3)

  • For JACKPOT accounts, extract ICP dimensions (vertical, size, geography, etc.)
  • Calculate composite ICP score
  • Assign ICP tier
  • Dashboard: Show ICP tier distribution within JACKPOT

Phase 3: Play Tagging (Week 3-4)

  • Tag accounts with plays (S1 from BuiltWith, S2 from marketing, S3 from Crossbeam)
  • Calculate intersection
  • Apply play multiplier
  • Dashboard: Show play intersection matrix

Phase 4: Campaign Execution (Week 4+)

  • Assign campaign type based on priority
  • Generate persona/job profile targets
  • Create messaging templates
  • Set up sequences in outreach tool
  • Track engagement and conversion

Summary & Recommendation

What We're Proposing

A four-layer cascading funnel that transforms thousands of partner tech accounts into a focused list of the highest-priority opportunities:

  1. Layer 1: Partner Tech Cohort - Filter by tech stack complexity (JACKPOT = CMS + Commerce + Hyperscaler)
  2. Layer 2: ICP Filter - Score against data-derived ICP (Fashion/Grocery + Enterprise + Elastic displacement)
  3. Layer 3: Sales Play Intersection - Apply play multipliers (Triple Play = S1 + S2 + S3)
  4. Layer 4: GTM Campaign - Assign ABM treatment (1:1, 1:Few, 1:Many)

Why This Approach

Without This Machinery

  • 2,800 accounts treated equally
  • Spray-and-pray outreach
  • Low response rates
  • Wasted AE time
  • Unclear prioritization

With This Machinery

  • 10-15 accounts get ABM 1:1
  • Data-driven prioritization
  • Higher response rates
  • Focused AE effort
  • Clear pipeline stages

The Ask

We request approval to implement this sales machinery as the foundation for Arian. The approach is:

  • Data-driven - Based on 1,848 customer evidence records
  • Systematic - Four clear layers with defined filters
  • Scalable - Can be automated and repeated
  • Measurable - Clear metrics at each stage
