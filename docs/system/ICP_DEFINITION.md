## 8. IDEAL CUSTOMER PROFILE (Composite)

### The "Perfect" Algolia Customer

| Dimension | Ideal Value | ICP Score Weight |
|-----------|-------------|------------------|
| **Vertical** | Fashion or Grocery | 25% |
| **Company Size** | Mid-Market to Enterprise ($50M-$5B) | 15% |
| **Geography** | US, UK, or EMEA | 10% |
| **Tech Stack** | Shopify/Magento/SFCC + Contentful/AEM | 15% |
| **Competitor** | Using Elastic, Solr, or SearchSpring | 20% |
| **Persona Access** | CTO or Director of Digital Experience | 15% |

### Composite ICP Score Formula

```
ICP_Score = (
    Vertical_Score × 0.25 +
    Size_Score × 0.15 +
    Geography_Score × 0.10 +
    TechStack_Score × 0.15 +
    Competitor_Score × 0.20 +
    Persona_Score × 0.15
)
```

### ICP Tier Thresholds

| Tier | Score Range | Label | Action |
|------|-------------|-------|--------|

| **Tier 3** | 50-69 | MODERATE | Nurture |
| **Tier 4** | 25-49 | WEAK | Low priority |
| **Tier 5** | 0-24 | ANTI-ICP | Exclude |

## 10. APPLICATION TO ARIAN

### ICP Layer × Target Data Layer

```
For each target account:

1. EXTRACT dimensions from target data:
   - Vertical (from BuiltWith industry, manual tagging)
   - Size (from SimilarWeb traffic, Yahoo Finance revenue)
   - Geography (from company HQ)
   - Tech Stack (from BuiltWith)
   - Competitor (from BuiltWith search tech detection)
   - Persona (from ZoomInfo contacts, hiring signals)

2. SCORE each dimension using ICP tables above

3. CALCULATE composite ICP score using weighted formula

4. APPLY ICP tier to prioritization:
   - Tier 1 accounts → HOT (pursue immediately)
   - Tier 2 accounts → WARM (active nurture)
   - Tier 3 accounts → COLD (low priority)
   - Tier 4-5 accounts → EXCLUDE or deprioritize
```

### ICP × Play Type Matrix

| Play Type | ICP Tier 1 | ICP Tier 2 | ICP Tier 3 | ICP Tier 4+ |
